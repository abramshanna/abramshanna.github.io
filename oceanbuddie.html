Background

This game was created during a 6-week school project by a team of 9 students, consisting of designers, artists and programmers. The game was shaped under the prompt "symbiosis". My main responsibility during this project was to manage the project, entailing anything from refining the core game idea, requesting assets and technical implementation to managing teams, communication and planning.

Gameplay

I helped shape the gameplay experience of Ocean Buddie by guiding the vision for what the game should feel like and how it should play. I spent the initial part of the project conceptualizing and exploring ideas, and I was responsible for ensuring a cohesive gameplay experience throughout the project.

Core Concept

The idea was to create a local co-op puzzle adventure game featuring two different characters, one on land and one in water. To play off the given theme of symbiosis the players are bound to their own element where they operate efficiently and with ease. The game is set in a tropical archipelago, with islands and the ocean as the two main elements. The environment and art direction strongly inspired and influenced the development of the game and its mechanics.

Co-op

The game is heavily focused on cooperation where the two players are dependent on each other in order to solve the puzzles and succeed.

Elements

The players are tied to their own unique element with their own strengths and limitations.

Puzzle

The different puzzles in the semi-open world introduces new mechanics and challenges along the way.

Gameplay Loop

The game is linear and starts at the beginning of the islands, where players are quickly introduced to their movement patterns and any platforming elements. As the pair makes their way through the level the puzzles are introduced one by one along with the individual mechanics. Puzzles get progressively more difficult and each of the puzzles are unique. If players do not cooperate they can not progress through the level and all puzzles have to be completed in order to reach the end of the game.

Movement and Camera

The idea was for movement to feel different for each character and was therefore adapted to their element. Swimmie is agile and can swim and jump high in the water, while Landie can run around on land, pick things up as well as climb.

The original idea was to implement a dynamic split screen camera, but failed due to technical difficulties. Instead it was settled on a static split camera and opted for a sidescroller style and layout in order for players to get a good overview of each puzzle and the environment, as well as to not loose the other player as easily.

Level Design

The puzzles and the level were planned, created and implemented by the level designers. I aided in preparation of the level assets, merging art assets with scripts and adding colliders. I assisted in drawing out the islands by hand, general scene dressing and setting up the menu scenes, as well as creating some visual effects. Playtesting of the puzzles was crucial to make sure that things were working as intended and to discover any bugs.

UI Design

I created the UI for the game by defining a fun and playful style. I created assets and implemented them in the engine.

In-game

The main goal of the in game UI was to convey the controls and mechanics to the players. The UI shows if the player is close to an intractable object. Hints for puzzles are provided to players through a prompt by the parrot plastie if the players walk up to him.

Menus

The menus were kept simple and a fade was implemented in between menus and the game for a soft transition. The player can pause the game and is provided with an end screen if they complete the game.

Colors

For a minimal design I used a white color for text and a very bright turquoise for backgrounds. I worked with shadows and opacity to make the text more readable. This was in hindsight a poor color choice and combination, especially against the bright game as the contrast was too low.

UI Scripting

After designing and implementing the UI I created the scripts to manage the functionality, such as playing audio, loading levels or showing the UI at the right time.

Interaction

Handles display of in-game UI for interactable objects.


public class UIInteraction : MonoBehaviour
{
    public Animator animator;
    GameObject child;
    private void Start()
    {
        child = this.transform.GetChild(0).gameObject;
    }
    
    private void OnTriggerEnter(Collider other)
    {
        if (other.tag == "Player")
        {
            child.SetActive(true);
            animator.SetTrigger("UIFadeIn");
        }
    }
    
    private void OnTriggerExit(Collider other)
    {
        if (other.tag == "Player")
        {
            animator.SetTrigger("UIFadeOut");
        }
    }
    public void UIFadeOutComplete()
    {
        child.SetActive(false);
    }
}

Hints

Handles display of in-game hints.


public class UIHint : MonoBehaviour
{
    public Animator animator;
    GameObject child;
    AudioSource audioSource;
    private void Start()
    {
        child = this.transform.GetChild(0).gameObject;
        audioSource = GetComponent();
    }

    private void OnTriggerEnter(Collider other)
    {
        if (other.tag == "Player")
        {
            audioSource.Play();
            child.SetActive(true);
            animator.SetTrigger("UIFadeIn");
        }
    }

    private void OnTriggerExit(Collider other)
    {
        if (other.tag == "Player")
        {
            animator.SetTrigger("UIFadeOut");
        }
    }
    public void UIFadeOutComplete()
    {
        child.SetActive(false);
    }
}

UI Audio

Handles audio related to the UI.


public class UIAudioManager : MonoBehaviour
{
    AudioSource[] audioSources;
    [SerializeField] AudioClip hoverSound;
    [SerializeField] AudioClip pressedSound;
    int currentScene;

    private void Awake()
    {
        audioSources = GetComponents();
        currentScene = SceneManager.GetActiveScene().buildIndex;
        Debug.Log(currentScene);
    }

    private void Start()
    {
        if (currentScene != 1)
        {
            audioSources[1].Play();
            audioSources[1].loop = true;
        }
        
    }

    public void hoverButton()
    {
        audioSources[0].PlayOneShot(hoverSound);
    }
    public void clickButton()
    {
        audioSources[0].PlayOneShot(pressedSound);
    }
}

UI Manager

Manages the UI, loading scenes and menus.


public class UIManager : MonoBehaviour
{
    public static UIManager instance;
    [SerializeField] Canvas pauseCanvas;
    [SerializeField] Animator animator;
    private int sceneToLoad;

    private void Awake()
    {
        if (instance == null)
        {
            instance = this;

        }
        else
        {
            GameObject.Destroy(this.gameObject);
            return;
        }
    }

    //fade to a scene
    public void FadeToScene(int levelIndex)
    {
        sceneToLoad = levelIndex;
        animator.SetTrigger("SceneFadeOut");
        Time.timeScale = 1f;
    }

    public void FadeComplete()
    {
        SceneManager.LoadScene(sceneToLoad);
    }


    //pause game - call on pause input
    public void PauseGame()
    {
        pauseCanvas.gameObject.SetActive(true);
        Time.timeScale = 0;
    }

    //resume game
    public void ResumeGame()
    {
        pauseCanvas.gameObject.SetActive(false);
        Time.timeScale = 1f;
    }

    //quit game
    public void QuitGame()
    {
        Application.Quit();
    }
}
